# Form 1 English Content Schema

## Purpose

This document defines a platform-ready `Form 1 English` curriculum structure for an AI-first, tutor-supported tuition system aligned to Malaysia lower secondary learning.

The goal is to support:

- AI tutoring
- mastery tracking
- adaptive revision
- tutor intervention
- parent visibility
- content operations

This is a product blueprint, not a textbook chapter list.

## Scope

Subject:

- English

Level:

- Form 1

Primary instructional model:

- AI handles daily reading practice, grammar drills, vocabulary work, and guided writing support
- Human tutors step in for explanation, speaking-through-writing coaching, motivation, and targeted intervention

---

## 1. Domain Map

| Domain Code | Domain Name | Why It Exists |
|---|---|---|
| `A` | Reading Comprehension | Build the ability to read short texts, find explicit meaning, infer simple ideas, and justify answers |
| `B` | Grammar and Usage | Stabilise sentence control, tense use, and basic English accuracy |
| `C` | Vocabulary and Language Awareness | Grow usable vocabulary through context clues, relationships, and precision |
| `D` | Writing Foundations | Move students from sentence-level output into coherent paragraph and guided writing |

---

## 2. Topic Register

| Topic Code | Topic Name | Summary |
|---|---|---|
| `A1` | Main Ideas and Explicit Information | Identify main ideas, supporting details, and directly stated information in short texts |
| `A2` | Inference and Supporting Evidence | Make simple inferences and justify them with clues from the text |
| `A3` | Reference and Context Clues | Resolve pronoun reference and infer meaning from surrounding language |
| `B1` | Parts of Speech and Sentence Roles | Recognise common word classes and their sentence functions |
| `B2` | Tenses and Subject-Verb Agreement | Control simple present, past, future, and accurate subject-verb matching |
| `B3` | Sentence Construction and Transformation | Build, improve, and transform sentences with clarity |
| `C1` | Vocabulary in Context | Use context clues to understand and apply unfamiliar words |
| `C2` | Word Choice and Relationships | Improve precision through synonyms, antonyms, collocations, and better word choice |
| `D1` | Sentence to Paragraph Writing | Turn linked ideas into short, coherent paragraphs |
| `D2` | Guided and Functional Writing | Produce short guided responses, messages, and practical writing tasks |

---

## 3. Mastery Design

Each topic is broken into `4` mastery nodes:

1. `Recognition`
2. `Procedure`
3. `Procedure / transfer`
4. `Application`

This keeps the English subject consistent with the platform-wide mastery model already used in mathematics.

### Example: `A2 Inference and Supporting Evidence`

- `IN-01` Recognise when an answer must be inferred
- `IN-02` Use text clues to make a simple inference
- `IN-03` Select evidence that supports an inference
- `IN-04` Explain an inference with evidence

### Example: `B2 Tenses and Subject-Verb Agreement`

- `TS-01` Recognise basic tense and agreement patterns
- `TS-02` Use correct tense in isolated sentences
- `TS-03` Maintain tense and agreement across linked sentences
- `TS-04` Edit tense and agreement errors in context

### Example: `D2 Guided and Functional Writing`

- `GW-01` Recognise task purpose and format
- `GW-02` Plan short guided responses
- `GW-03` Write messages and simple functional texts
- `GW-04` Deliver a clear task-fit written response

---

## 4. Recommended Learning Flow

For daily study, the AI tutor should sequence English like this:

1. Warm-up reading or language check
2. Micro lesson on the current mastery node
3. Guided practice
4. Short independent check
5. Revision recommendation
6. Tutor escalation if needed

### AI tutor should prioritise:

- explicit reading before inference
- sentence accuracy before paragraph fluency
- contextual vocabulary before isolated memorisation
- task purpose before guided writing output

---

## 5. Common Misconception Families

The current Form 1 blueprint seeds these high-value misconception groups:

- `detail-instead-of-main-idea`
- `guess-without-evidence`
- `pronoun-reference-drift`
- `tense-switch-without-reason`
- `fragment-and-run-on-mixup`
- `dictionary-meaning-overload`
- `imprecise-word-choice`
- `paragraph-without-focus`
- `format-purpose-mismatch`

These are designed to drive:

- AI feedback wording
- tutor intervention rules
- parent-friendly explanations

---

## 6. Tutor Intervention Points

Tutors should be alerted when students show patterns like:

- repeated inference answers without evidence
- unstable tense control across short writing
- frequent sentence fragments or run-ons
- weak paragraph focus despite guided prompts
- task responses that ignore audience or format

Tutor actions inside the platform should include:

- assign mini live intervention
- unlock remedial grammar or writing path
- leave parent-visible support note
- reduce task scope and rebuild fundamentals

---

## 7. Parent Visibility

For English, parent-facing reports should emphasise:

- reading comprehension progress
- grammar stability
- writing confidence
- current weak area
- next focus

### Example parent summary

- Current focus: `Inference and Supporting Evidence`
- Improved this week: `Main Ideas and Explicit Information`
- Needs support: `Tense consistency in paragraph writing`
- Next step: `Use evidence sentences in reading responses`

---

## 8. MVP Build Order

Recommended build order for authored English content:

1. `A1 Main Ideas and Explicit Information`
2. `A2 Inference and Supporting Evidence`
3. `B2 Tenses and Subject-Verb Agreement`
4. `B3 Sentence Construction and Transformation`
5. `D1 Sentence to Paragraph Writing`
6. `D2 Guided and Functional Writing`
7. `A3 Reference and Context Clues`
8. `C1 Vocabulary in Context`
9. `C2 Word Choice and Relationships`
10. `B1 Parts of Speech and Sentence Roles`

This order gets the platform to a useful teaching core faster:

- reading
- grammar
- paragraph writing

before expanding the full vocabulary and language-awareness layer.

---

## 9. Summary

`Form 1 English` in this platform should not be a flat list of grammar drills.

It should behave like a structured mastery system:

- 4 domains
- 10 topics
- 40 mastery nodes
- linked misconception rules
- tutor intervention hooks
- parent-visible progress language

This lets English use the same curriculum engine already powering the mathematics pipeline.
