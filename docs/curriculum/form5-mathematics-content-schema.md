# Form 5 Mathematics Content Schema

## Purpose

This document extends the mathematics curriculum architecture into `Form 5` for a Malaysia KSSM-aligned, AI-first, tutor-supported tuition workflow.

The goal is to support:

- adaptive AI tutoring
- mastery-node progression
- node-based homework generation
- tutor intervention routing
- parent-visible progress summaries
- curriculum content operations

This is a platform schema, not a textbook chapter transcript.

## Scope

Subject:

- Mathematics

Level:

- Form 5

Instructional model:

- AI handles daily lessons, practice, and checkpoint flow
- Human tutors step in for diagnosis, live explanation, intervention, and parent communication

## 3. Mastery Node Design Pattern

Each Form 5 topic is broken into node-level progression:

1. `Recognition`
2. `Procedure`
3. `Application`
4. `Retention` where needed later

### Example: `A1 Variations`

- `VA-01` Recognise direct, inverse, and joint variation structure
- `VA-02` Form variation equations
- `VA-03` Find constants and unknown values in variation
- `VA-04` Use variation in practical relationships

### Example: `D2 Mathematical Modelling`

- `MM-01` Recognise assumptions and variables in a model
- `MM-02` Translate a situation into a mathematical model
- `MM-03` Evaluate and refine a mathematical model
- `MM-04` Use modelling to justify a recommendation

---

## 4. Cross-Topic Dependency Logic

Important recommended bridges in Form 5:

- `Variations -> Matrices`
- `Congruence -> Enlargement`
- `Enlargement -> Combined Transformations`
- `Trigonometric Ratios (Form 3) -> Trigonometric Functions and Graphs`
- `Measures of Dispersion (Form 4) -> Measures of Dispersion for Grouped Data`
- `Variations + Trigonometric Graphs -> Mathematical Modelling`
- `Financial Management (Form 4) -> Insurance and Taxation`

These do not replace within-topic mastery edges. They help the platform recommend revision paths and tutor follow-up.

## 5. Misconception Design

Form 5 currently seeds misconception clusters for:

- variation form mismatch
- matrix order blindness
- congruence condition swap
- scale-factor direction gap
- transformation order loss
- trig graph feature mix-up
- grouped-data midpoint drift
- hidden modelling assumptions
- consumer-choice justification gap

These are used for:

- tutor risk surfacing
- AI hint ladder selection
- homework follow-up language
- parent-facing weak-topic summaries

---

## 6. Node-Driven Content Objects

For each mastery node, the platform seeds:

- `1` lesson module
- `1` checkpoint structure
- `1` intervention rule
- `1` revision rule
- `5` question pools
  - teach
  - guided practice
  - diagnostic
  - mastery
  - revision
- placeholder question items for each pool

This keeps Form 5 aligned with the same content model already used for Forms 1 to 4.
